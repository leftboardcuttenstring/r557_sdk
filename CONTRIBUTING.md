## How to Contribute

### Reporting Issues
- Before opening a new issue, review existing ones to avoid duplicates.
- Provide a clear title, description, steps to reproduce the issue, and any relevant logs or screenshots.

### Submitting Pull Requests
1. Fork the repository and create a new branch for your changes (e.g., `feature/new-feature` or `fix/bug-fix`).
2. Make your changes, ensuring they comply with the project's coding style (this is optional, but will ultimately result in less confusion in the code).
3. Test your changes thoroughly.
4. Commit your changes with clear and informative messages.
5. Open a pull request for the main branch, including:
- A description of what the pull request does.
- Links to any related issues (e.g., "Closes issue "[issue name]"").
- Confirmation that your contribution is licensed under the MPL 2.0 (leave any required labels).

By submitting a pull request, you agree that your contributions are licensed under the MPL 2.0, allowing the project to incorporate them appropriately. If your contribution includes third-party code, ensure it is MPL 2.0-compatible and cite the source if necessary.
